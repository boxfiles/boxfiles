# Liquid property templates for context facts

Boxfiles will expose context facts in manifests as Liquid property access such as `{{ user.home }}` while keeping `ContextService` as the fact ledger. Context fact paths used in templates must be declared up front as Liquid-safe dotted identifiers, top-level context namespaces are single-owner and reserved against template shadowing, facts resolve at planning time with per-plan stability, and missing or cyclic facts fail planning with diagnostics. We chose Liquid-native bare property syntax over `ctx.user.home` or `{{ "user.home" | fact }}` for manifest ergonomics, accepting that a spike must prove true lazy reads; if LiquidJS cannot support that, phase 1 may eagerly precompute all declared facts as a documented fallback.

Phase 1 will render only action `with` string values and manifest/step `when` templates, gate false `when` before provider planning, include skipped-step records in plans, allow built-in Liquid filters and raw blocks, disable file includes/partials/layouts, and treat manifests as trusted local config. Providers keep raw `ctx.facts` compatibility for now, but that path is deprecated in favor of the new context access model.
